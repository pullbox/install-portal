To install a Collector without Root Access:
                
    
                  
  1. From get.takipi.com/takipi-latest.tar.gz, download the Collector installation TAR file.
    2. 
                  
  2. Extract files to your working folder:
                    tar -xf takipi-latest.tar.gz
                  A ‘takipi’ folder is created.
    3. 
                  
  3. If you do not possess an Installation key, from the OverOps web application, generate an installation key.
    4. 
                  
  4. Copy the installation key and paste it into a text file named installation.key.
    5. 
                  
  5. Save the installation.key file in the /takipi/work directory on your machine.
    6. 
                  
  6. According to the template below, create the collector.properties file:
                      
                        jvmPath=${JAVA_HOME}/jre/lib/amd64/server/libjvm.so
                        baseUrl=https://backend.takipi.com/
                        s3ConnectionTestUrl=https://s3.amazonaws.com/app-takipi-com/ConnectionTest
                        # The TCP port to listen on
                        listenOnPort=
                        # Used by both Collector & Agent, Arbitrary token used to identify the Agent against the Service
                        passphrase=
                        # The server name of the Collector. Leave empty for HOSTNAME
                        serverName=
                        # The internal JVM max heap size
                        javaHeapSize=1G
                      ...
                  
    7. 

                  
  7. Place the properties file in the takipi directory on your machine.
    8. 
                  
  8. Run the Collector:
                  /takipi/bin/takipi-service
    9. 
                  
  9. Test the installation:From the OverOps web application (app.overops.com) Dashboard, click Add Server and then, from the dialog box, click Next and then click Test Installation.
                      When the connection is established, the Collector status is 'Monitoring'.
    10. 
                  
  10. Test that your application throws exceptions.
                      Rootless installation does not enable the Collector to start automatically after a reboot.
    11. 
                  
  11. Run the Collector as a daemon according to your system.
    12.

Installing the Storage Server on a Local Server

                    From the following URL, download the Storage Server installation file:
                    wget https://app-takipi-com.s3.amazonaws.com/deploy/takipi-storage/takipi-storage-latest.tar.gz

                    Extract files to the /opt directory:

                    sudo tar zxvf takipi-storage-latest.tar.gz -C /opt

                    Configure and copy the takipi-storage file to the init.d directory:

                    sudo chmod +x /opt/takipi-storage/etc/
                    sudo cp /opt/takipi-storage/etc/takipi-storage.init /etc/init.d 

                    Verify that the /etc/init.d/takipi-storage
                    file points to a valid Java installation.
                    Depending on the Linux distribution, run installation:
                    Ubuntu:
                    sudo /usr/sbin/update-rc.d takipi-storage.init defaults
                    Other Linux:
                    sudo /sbin/chkconfig takipi-storage on
                    Start the service:
                    sudo service takipi-storage start
                    Verify that the Storage Server is running:
                    ps -ef | grep takipi-storage
                    If service does not start check the logs and send to the OverOps team:
                    less /opt/takipi-storage/log/takipi-storage.log.
